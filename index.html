[alias]
    lg = log --graph --abbrev-commit --decorate --date=relative --format=format:'%C(bold blue)%h%C(reset) - %C(bold green)(%ar)%C(reset)%C(bold yellow)%d%C(reset) %C(white)%s%C(reset) %C(dim white)- %an%C(reset)'
    lgn = !"git lg -n"
    lga = !"git lg --all"
    lgan = !"git lg --all -n"

Local vs remote

in LOCAL repo:   play or mess up == fun !

in REMOTE repo:   play or mess up == instant pain !

however, git is distributed, and we can fix it

Dos and don'ts

please do ~vs~ please do not

Do

use gitignore.io

have a clean working directory before doing anything else

branch out, play with local repo

Think twice before pushing

Be aware of the repo and communicate!

Do not

...panic

"please don't" reminder

do NOT put generated code into repo
do NOT put binaries into repo
do NOT put personal IDE setup into repo
do not put credentials into repo

DO NOT GIT PUSH --FORCE
DO NOT GIT COMMIT --AMEND remote commits
DO NOT GIT REBASE remote commits
DO NOT GIT PUSH --ALL

...basically do not destroy or otherwise alter remote history.

A bit about merge and rebase

safe rebase ~ when to merge vs rebase

Lossless rebase 1/2


kd@feature$ git branch -f last_rebase
kd@feature$ git rebase master
# if I messed up
kd@feature$ git reset --hard last_rebase
# profit?

or


kd@feature$ git checkout -b rebase_temp
kd@rebase_temp$ git rebase master
# if I messed up
kd@rebase_temp$ git checkout feature
kd@feature$ git branch -D rebase_temp
# profit?

Lossless rebase 2/2

its not breaking the rebase process that gets you
(detached HEAD state)

it is forcing it to to the end in undesirable state!

When to merge vs rebase

please do $git pull --rebase origin for minor fixes

always merge in feature branches

always merge if rebase would alter remote history!
